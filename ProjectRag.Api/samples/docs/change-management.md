# Change Management

Production changes must be documented before deployment.

Each change record should include the summary, risk level, rollout plan, rollback plan, owner, expected customer impact, and validation steps.

High-risk changes require approval from engineering leadership and must be scheduled outside peak usage windows.

Emergency changes may bypass standard approval when needed to restore service or address an active security incident. Emergency changes must be reviewed afterward.

Deployment owners must monitor health checks, logs, and customer impact after release.
